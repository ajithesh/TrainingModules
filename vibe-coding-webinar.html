🚀 The Rise of Vibe Coding: Adapt or Get Left Behind

▼

Understanding the paradigm shift — from typing code to describing intent

Coined by Andrej Karpathy (Feb 2025) Core Idea Describe, don't type Powered by LLMs & AI Agents Shift Code → Intent

What is Vibe Coding?

Andrej Karpathy's definition: "There's a new kind of coding I call vibe coding, where you fully give in to the vibes, embrace exponentials, and forget that the code even exists. It's not really coding — I just see stuff, say stuff, run stuff, and copy-paste stuff, and it mostly works."

Vibe Coding is a style of programming where you describe what you want in natural language and let AI tools generate the code for you. Instead of manually writing every function, class, and variable, you collaborate with AI — you set the direction, the AI writes the code, and you iterate together until it's right.

🖥️ Traditional Coding

  • You think about the logic
  • You write every line of code
  • You debug by reading stack traces
  • You search Stack Overflow for solutions
  • You spend hours on boilerplate

Speed: Hours to days for a feature. You are the typist AND the architect.

🎨 Vibe Coding

  • You describe the intent
  • AI generates the code
  • You review, iterate, refine
  • You prompt for fixes and improvements
  • Boilerplate is instant

Speed: Minutes to hours for a feature. You are the architect, AI is the typist.

What Does a Real Prompt Look Like?

Building a real app is about chaining prompts — each one building on the last. Notice how simple the language is — no jargon needed:

// Prompt chain for building a Placement Prep Tracker:

// 1. Start the project
"Build me a Placement Prep Tracker website where I can
 track which companies I'm applying to during campus
 placements. I want a dashboard showing my stats and a
 page to manage all my company applications. Use a
 modern dark theme."

// 2. Set up the database
"I need to save my data. Create a database table to store
 each company I'm tracking — save the company name, job role,
 salary package, application deadline, current status, which
 interview round I'm at, and any personal notes."

// 3. Build the main tracker
"Show my companies as cards organized in columns:
 Wishlist, Applied, Interviewing, Offer, Rejected.
 I should be able to drag a company card from one
 column to another when my status changes."

// 4. Add a way to add new companies
"Add a button that opens a form where I can add a new
 company. I should fill in the company name, role, salary,
 deadline, and any notes. Show a success message when saved."

// 5. Make it live on the internet
"Deploy this so I can access it from my phone too.
 Give me a link I can share with friends."

💡 Notice: Not a single technical term in those prompts. No "Next.js", no "Supabase", no "environment variables." The AI figures out the tech — you just describe what you want like you're talking to a friend. That's the whole point of vibe coding.

🗄️ Setting Up Supabase (Database & Backend)

💡 What is Supabase? An open-source Firebase alternative that gives you a PostgreSQL database, instant REST API, authentication, and real-time subscriptions — all with a generous free tier. No backend code needed.

Step 1: Create a Supabase Project

1️⃣

Sign up at supabase.com

Free tier — no credit card required. Sign in with GitHub for fastest setup.

2️⃣

Click "New Project"

Pick a name (e.g., "VibeCoding"), choose a region close to you, set a database password.

3️⃣

Wait ~30 seconds for provisioning

Supabase sets up your PostgreSQL database, REST API, and auth system automatically.

Step 2: Get Your API Keys

🔑 Where to find them: Go to Settings → API Keys in your Supabase dashboard. You need two values:

Key Where to Find What It's For Safe to Share?
Project URLSettings → General → Project ID
https://<project-id>.supabase.co		The API endpoint for your database✅ Yes (public)
Anon / Public KeySettings → API Keys → anon publicClient-side access with Row Level Security✅ Yes (public, RLS enforced)
Service Role KeySettings → API Keys → service_role secretServer-side admin access, bypasses RLS❌ NEVER expose this

Step 3: Create the .env.local File

In your project root, create a file called .env.local — this is where your app reads the keys from:

# .env.local — put this in your project root
# ⚠️ Add .env.local to .gitignore so it never gets committed!

NEXT_PUBLIC_SUPABASE_URL=https://your-project-id.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-public-key-here

⚠️ Important: The NEXT_PUBLIC_ prefix means these variables are exposed to the browser. That's fine for the anon key because Supabase uses Row Level Security (RLS) to control access. Never put the service_role key in a NEXT_PUBLIC_ variable.

Step 4: Run the SQL Migration

Go to SQL Editor in your Supabase dashboard (left sidebar), paste your migration SQL, and click Run:

-- Create the applications table (Placement Prep Tracker)
create table if not exists applications (
  id uuid default gen_random_uuid() primary key,
  company_name text not null,
  role text not null,
  package text,
  deadline date,
  status text default 'Wishlist',
  round text default 'Not Started',
  notes text,
  created_at timestamptz default now()
);

-- Enable Row Level Security
alter table applications enable row level security;

-- Allow anyone to read
create policy "Anyone can read"
  on applications for select using (true);

-- Allow anyone to insert (demo only)
create policy "Anyone can insert"
  on applications for insert with check (true);

-- Allow anyone to update (demo only)
create policy "Anyone can update"
  on applications for update using (true);

Step 5: Connect Your App to Supabase

The Supabase client code (AI generates this for you):

// lib/supabase.ts
import { createClient } from "@supabase/supabase-js";

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!;
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!;

export const supabase = createClient(supabaseUrl, supabaseAnonKey);

🚀 Deploying to Vercel (Go Live in 2 Minutes)

💡 What is Vercel? A cloud platform built for frontend & Next.js apps. Push your code → it's live on the internet with HTTPS, CDN, and auto-deploys. Free tier is very generous — no credit card needed.

Deployment Steps

1️⃣

Push code to GitHub

Create a repo and push. Make sure .env.local is in .gitignore — never commit secrets to GitHub!

2️⃣

Sign in to vercel.com with GitHub

Free Hobby plan — unlimited deployments, custom domains, HTTPS included.

3️⃣

Click "Add New Project" → Import your repo

Vercel auto-detects it's a Next.js app and configures build settings for you.

4️⃣

Add Environment Variables (before clicking Deploy)

This is the critical step — Vercel needs your Supabase keys to connect to the database.

5️⃣

Click Deploy → Live in ~30 seconds

Vercel gives you a URL like your-app.vercel.app. Share it with anyone!

Environment Variables in Vercel

On the deployment screen, expand "Environment Variables" and add these two:

Key Value
NEXT_PUBLIC_SUPABASE_URLhttps://<your-project-id>.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY<your-anon-public-key>

⚠️ Without these env vars, the deployed app will crash. This is the #1 mistake — the app works locally (reads from .env.local) but breaks on Vercel because the variables aren't set there.

After Deployment — Auto-Deploys

Every time you git push to GitHub, Vercel automatically rebuilds and redeploys your app. No manual steps needed — push code, it's live in seconds.

Concern Reality

Productivity Boosting Tips

✅ What Works

  • Be specific in prompts — "Add a Kanban board with 5 columns: Wishlist, Applied, Interviewing, Offer, Rejected" beats "add a tracker"
  • Iterate fast — don't spend 10 min crafting the perfect prompt, just start and refine
  • Use screenshots — show AI what you want by referencing existing designs (like Trello)
  • Chain prompts — build feature by feature, not everything at once

❌ What Doesn't Work

  • Vague prompts — "make it better" gives random results
  • Massive prompts — asking for 20 features at once overwhelms the AI
  • Zero review — never ship AI code without reading it
  • Ignoring errors — if the AI's code has bugs, describe the bug clearly instead of re-prompting from scratch

Anticipated Q&A — Presenter Cheat Sheet

❓ "Is this going to replace software engineers?"

No. It's going to change what software engineers do. Just like Excel didn't replace accountants, AI won't replace developers — but developers who use AI will replace those who don't. The role evolves from "code writer" to "solution architect + AI operator."

❓ "How do I know if the AI-generated code is good?"

Same way you evaluate any code: read it, test it, review it. Tools like Copilot show you the code — you can inspect every line. For production, always add tests, do code reviews, and run security scans. Over time, you develop intuition for what good AI output looks like.

❓ "What about complex enterprise applications?"

Vibe coding works best for greenfield projects and MVPs. For complex enterprise systems with legacy code, strict compliance, and custom infrastructure, AI assists but doesn't replace experienced engineers. It accelerates the 80% that's routine so humans can focus on the 20% that's complex.

❓ "Which tool should I start with?"

If you're a developer: Start with GitHub Copilot (free in VS Code) — it integrates into your existing workflow. If you're non-technical: Start with Bolt.new or Lovable — browser-based, no setup needed, visual results instantly.

❓ "Is the code AI generates secure?"

Not always — AI can introduce vulnerabilities just like human developers can. Always review security-critical code (authentication, payments, data handling). Use automated security scanning tools. Never deploy AI-generated code handling sensitive data without thorough review.
